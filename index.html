学习使用git以及相关命令

这个文本是为了练习使用git建立的

        git部分命令介绍：
        /***** 初始化本地仓库 *****/
        $ git init 初始化本地仓库

        /***** 往git版本库中添加文件 *****/
        $ git add [filename] 添加一个文件到暂存区
        $ git commit -m 'message' 提交代码到本地仓库当前分支，'message'为本次提交版本描述，最好描述简单明了，便于他人阅读

        $ git add . 添加目录下的所有文件到暂存区

        $ git status 查看状态
        $ git diff 查看不同

        /***** 回退版本 *****/
        $ git reset --[hard/soft/]
        $ git log 查看提交历史
        $ git reflog 查看所有命令历史

        /***** 撤回修改 *****/
        $ git checkout -- [filename] 撤回工作区的所有修改（未add前）
        $ git reset HEAD [filename] 可以把暂存区的修改撤销掉（unstage），重新放回工作区（在commit之前）
    
        /***** 添加远程库 *****/
        
        $ git remote add origin [git项目地址] 例如：我的https://github.com/stt0911/myDemo.git
        
        $ git push [-u] origin master 将本地库内容推送到远程库上，第一次推送需要加上-u
    
        /***** 创建与合并分支 *****/
        $ git branch [branchname] 创建分支
        $ git checkout [branchname] 切换分支
        $ git checkout -b [branchname] 命令加上-b参数表示创建并切换
        
        $ git switch -c [branchname] 创建并切换分支
        $ git switch [branchname] 切换分支

        /****** 合并分支 ******/
        $ git merge [branchname] 合并指定分支到当前分支
        $ git branch -d [branchname] 删除分支
        
        /***** 分支管理策略 ******/
        合并分支时，git默认使用Fast forward模式，这种模式删除分支后，会丢掉分支信息。
        如果强制禁用Fast forward模式， git会在merge时生成一个新的commit，这样从分支历史上就可以看到分支信息。
        $ git merge --no-ff -m "merge with no-ff" [branchname]
    
        /***** Bug分支 *****/
        $ git stash 把当前工作现场“储藏”起来，等以后恢复现场后继续工作
        $ git stash list 获取stash内容
        
        $ git stash apply 恢复隐藏的工作现场
        $ git stash drop 删除stash内容

        $ git stash pop 恢复同时删除stash内容，相当以上两个命令
        可以多次stash，恢复的时候，先用git stash list查看，然后恢复指定的stash，使用下面的命令：
        $ git stash apply stash@{0}
    
        $ git remote -v 查看远程库信息
        $ git push origin branchname 推送本地分支到远程仓库
        推送失败，先用git pull抓取新提交
        $ git checkout -b branchname origin/branchname  在本地创建和远程分支对应的分支

        /***** 多人协作 *****/
        并不是一定要把本地分支推送到远程，哪些分支需要推送，哪些不需要呢？
            1、master分支是主分支，因此需要时刻与远程同步；
            2、dev分支是开发分支，团队所有成员都需要在上面工作，所以也需要与远程同步；
            3、bug分支只用于本地修复bug，没有必要推送到远程；
            4、feature分支是否推送，取决于跟小伙伴是否有合作


        /***** 多人习作模式 ******/
        多人协作的工作模式通常是这样：
            首先，可以试图用git push origin[branch-name]推送自己的修改；
            如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；
            如果合并有冲突，则解决冲突，并在本地提交；
            没有冲突或者解决掉冲突后，再用git push origin [branch-name]推送就能成功！
            如果git pull提示no tracking information，则说明本地分支和远程分支的链接关系没有创建，用命令git branch --set-upstream-to [branch-name] origin/[branch-name]。
        这就是多人协作的工作模式
